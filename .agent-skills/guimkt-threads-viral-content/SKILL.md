# Skill: Threads Viral Content — Conteúdo Viral Ético para Threads

## Propósito

Gerar conteúdo para Threads que **viraliza pela substância, não pela manipulação**.
A mecânica viral é cientificamente embasada (dados de 307+ posts virais analisados).
A camada ética é inegociável e hardcoded em cada output.

O resultado: conteúdo que para o scroll, gera engajamento real e constrói autoridade
profissional — sem inventar histórias, fabricar dados ou manipular emoções.

**Antes de gerar qualquer conteúdo, leia os arquivos de referência relevantes:**
- `references/diretrizes-eticas.md` — 10 diretrizes éticas obrigatórias (LEIA SEMPRE)
- `references/templates-threads.md` — Templates para cada formato de Thread
- `references/templates-posts.md` — Templates para cada formato de Post
- `references/dados-viralidade.md` — Dados de performance e cheat-sheet de viralidade
- `references/hook-archetypes.md` — 10 arquétipos de hook com templates plug-and-play

---

## Dois Modos de Operação

### THREAD (conteúdo longo, multi-post)
- 8-14 posts individuais de até 500 caracteres cada
- Estrutura obrigatória: **Hook → Value → TLDR → Pergunta → CTA**
- 700-900 palavras no total
- Subtipos: Professor, Narrador, Sintetizador, Analista, Auto

### POST (conteúdo curto, single post)
- 1 post de até 500 caracteres
- Estrutura: **Hook + Insight em um único bloco**
- Subtipos: One-Liner, Listicle, Listicle+CTA, Provocação, Auto

## Processo de Criação (Obrigatório)

### Etapa 1 — Coletar Inputs

Antes de gerar, o agente DEVE ter:

| Input | Obrigatório? | Como obter |
|-------|-------------|------------|
| **Tema/Tópico** | Sim | Usuário fornece |
| **Formato** | Sim | Usuário escolhe ou Auto (agente decide) |
| **Subtipo** | Não | Se não especificado, agente escolhe o melhor |
| **Dados biográficos** | Sim, se template pede | Perguntar ao usuário — NUNCA inventar |
| **Experiência/case real** | Sim, se template pede | Perguntar ao usuário — NUNCA inventar |
| **Oferta/CTA** | Não | Usar se disponível, omitir se não |
| **Tom desejado** | Não | Default: provocador-técnico. Usuário pode ajustar |

**REGRA DE OURO**: Se um template exige dados pessoais (idade, anos de experiência,
história de transformação) e o usuário não forneceu, PERGUNTE. Nunca preencha com
ficção. Se o usuário não quiser fornecer, use um template que não exija dados pessoais.

### Etapa 2 — Selecionar Template

Consulte `references/templates-threads.md` ou `references/templates-posts.md`
conforme o modo escolhido. Cada template tem:
- Estrutura (Hook / Hold / Help)
- Regras específicas
- Exemplos de referência
- Variações de tom

### Etapa 3 — Gerar Conteúdo

Aplique simultaneamente:
1. O template selecionado
2. As diretrizes éticas de `references/diretrizes-eticas.md`
3. Os dados de viralidade de `references/dados-viralidade.md`
4. Os dados reais do usuário (nunca inventados)

### Etapa 4 — Verificação Ética (OBRIGATÓRIA)

Antes de entregar o output, o agente DEVE rodar este checklist mental:

```
□ Alguma história foi inventada?                    → Se sim, REMOVER
□ Alguma estatística não tem fonte?                  → Se sim, QUALIFICAR ou REMOVER
□ Algum claim está fora do domínio do autor?         → Se sim, SINALIZAR
□ Algum FOMO é fabricado (escassez/urgência falsa)?  → Se sim, REMOVER
□ Dados biográficos vieram do usuário?               → Se não, PERGUNTAR
□ CTA promete algo que existe e é entregável?        → Se não, AJUSTAR
□ Tom é provocador MAS fundamentado?                 → Se não, FUNDAMENTAR
```

### Etapa 5 — Entregar com Estrutura

O output DEVE ser entregue assim:

**Para Threads:**
```
POST 1 — HOOK
[conteúdo]

---

POST 2 — [TAG ESTRUTURAL]
[conteúdo]

---

POST 3 — [TAG ESTRUTURAL]
[conteúdo]

[...continua...]

---

POST N — CTA
[conteúdo]
```

Tags estruturais válidas: HOOK, AUTHORITY, VALUE, CONTRAST, PAYOFF, TLDR, QUESTION, CTA

**Para Posts:**
```
[conteúdo do post]

---
Formato: [subtipo]
Caracteres: [xxx/500]
```

## Subtipos de Thread — Quando Usar Cada Um

| Subtipo | Quando usar | Estrutura dominante |
|---------|-------------|-------------------|
| **Professor** | Usuário quer ensinar processo, framework ou método | Hook → Passos numerados → TLDR → CTA |
| **Narrador** | Usuário tem história real de transformação para contar | Hook biográfico → Lições de vida → Reflexão → CTA |
| **Sintetizador** | Usuário quer curar insights de múltiplas fontes | Hook com promessa → Insights condensados → TLDR → CTA |
| **Analista** | Usuário quer analisar dados, tendências ou benchmarks | Hook com dado surpreendente → Análise → Conclusão → CTA |

### Regras por Subtipo

**Professor:**
- Cada lição = 2 frases (O QUE fazer + POR QUE funciona)
- Ordenar da mais contrarian para a mais convencional
- Incluir pelo menos 1 exemplo prático real

**Narrador:**
- Hook DEVE ter: idade + tempo de experiência + dor + transformação
- TODOS os dados biográficos vêm do usuário (perguntar se não tiver)
- Vulnerabilidade real, não performática

**Sintetizador:**
- Citar fontes reais (artigos, livros, reports, dados de mercado)
- Não inventar citações. Se não há fonte, é opinião — declare como tal
- Conectar insights a ações práticas

**Analista:**
- Dados DEVEM ser reais e verificáveis (perguntar ao usuário)
- Se o dado é estimativa, declarar como tal
- Incluir contexto para os números (comparação, benchmark, tendência)

## Subtipos de Post — Quando Usar Cada Um

| Subtipo | Quando usar | Tamanho ideal |
|---------|-------------|---------------|
| **One-Liner** | Provocação rápida, opinião forte, verdade desconfortável | 1-3 linhas, < 150 chars |
| **Listicle** | Lista de valor rápido, dicas, coisas que aprendi | 5-10 itens, até 500 chars |
| **Listicle+CTA** | Listicle que conecta com oferta do usuário | Listicle + 2-3 linhas de CTA |
| **Provocação** | Hot take, contraste de crenças, "X é tóxico" | 2-6 linhas, punch máximo |

## Regras Globais de Formatação (TODAS as threads e posts)

### Obrigatório
- Nível de leitura: 4º-5º ano (FK ≤ 5). Frases curtas, palavras simples
- Máximo 25 palavras por frase
- Máximo 3 linhas por parágrafo
- Whitespace liberal entre blocos de texto
- Sem hashtags no output
- Máximo 3 emojis por post/thread inteira
- NUNCA usar 🚀 ou 🔥 (marcadores de conteúdo genérico)
- Emojis numerados (1️⃣ 2️⃣ 3️⃣) são aceitáveis para listas
- Emojis direcionais (👇 ➡) são aceitáveis para CTAs, com parcimônia

### Proibido
- Tom de IA genérica ("No mundo atual...", "Neste artigo vamos...")
- Clichês motivacionais ("Acredite no seu potencial", "Saia da zona de conforto")
- Corporate speak ("Sinergias", "Agregar valor", "Stakeholders")
- Filler ("Na verdade", "Basicamente", "Literalmente" sem função)
- Estruturas de pergunta retórica seguida de resposta ("Sabe o que é? É que...")

### Encorajado
- Verbos de alto impacto (ver banco em `references/dados-viralidade.md`)
- Contraste direto (X ≠ Y, "Não é X. É Y.")
- Metáforas visuais concretas
- Ritmo de frases que alterna curto-longo-curto
- Punchlines que fecham com impacto (última frase mais curta que as anteriores)

## Geração a Partir de URL

Quando o usuário fornecer uma URL como input:

1. **Extrair** os insights/argumentos principais do conteúdo fonte
2. **Reinterpretar** com a voz e expertise do usuário (NUNCA copiar)
3. **Creditar** a fonte original quando relevante ("Inspirado por [fonte]")
4. **Adaptar** para formato Threads seguindo os templates desta skill

Fontes aceitáveis: artigos de blog, posts do Reddit, posts do LinkedIn, reports,
newsletters, transcrições de podcast/vídeo.

O output NUNCA é cópia. É sempre reinterpretação original.

## Adaptação de Tom

O tom default é **"Estrategista Provocador"**: autoridade técnica com irreverência
calculada. Mas o agente deve adaptar conforme o nicho do usuário:

| Nicho do usuário | Adaptação de tom |
|-----------------|-----------------|
| Marketing B2B / Performance | Provocador-técnico (default). Jargão de mercado liberado |
| SaaS / Tech | Mais analítico, menos provocador. Dados > opinião |
| Coaching / Desenvolvimento pessoal | Mais empático, vulnerável. Menos agressivo |
| Finanças / Investimentos | Cauteloso com claims. Disclaimers quando necessário |
| Criadores de conteúdo / Creators | Mais casual, irreverente. Referências pop liberadas |
| Saúde / Bem-estar | Máxima cautela. Nunca afirmar sem base científica |

Para qualquer nicho: as 10 diretrizes éticas são inegociáveis e não se adaptam ao tom.

## Exemplo de Output Completo (Thread — Subtipo Professor)

> **Input do usuário:** "Cria uma thread sobre por que a maioria das landing pages
> não converte. Sou especialista em CRO com 10 anos de experiência."

```
POST 1 — HOOK
10 anos otimizando landing pages me ensinaram uma verdade desconfortável:

A maioria não converte porque foi construída pra agradar o dono, não o visitante.

Aqui vão 7 erros que eu vejo toda semana — e o que fazer em vez disso:

---

POST 2 — VALUE
1️⃣ Hero sem clareza

O visitante tem 3 segundos.

Se o headline não responde "o que é, pra quem é, por que importa" — ele saiu.

---

POST 3 — VALUE
2️⃣ CTA genérico

"Saiba mais" não é um CTA. É um convite pra fechar a aba.

Diga exatamente o que acontece quando clicam.

---

POST 4 — VALUE
3️⃣ Prova social decorativa

"Mais de 500 clientes satisfeitos" não convence ninguém.

"Reduzimos o CAC do cliente X em 37% em 90 dias" — isso convence.

---

POST 5 — VALUE
4️⃣ Formulário longo demais

Cada campo extra é uma razão pra desistir.

Nome + email + telefone. O resto você pergunta depois.

---

POST 6 — VALUE
5️⃣ Design acima de conversão

A LP mais bonita que já vi convertia 0,3%.

A mais feia? 12%. Clareza > estética. Sempre.

---

POST 7 — VALUE
6️⃣ Sem urgência real

Urgência fabricada = manipulação.

Urgência real = "vagas limitadas porque eu atendo 5 clientes por mês."

A diferença? Uma constrói confiança. A outra destrói.

---

POST 8 — VALUE
7️⃣ Ignorar mobile

60%+ do tráfego BR é mobile.

Se sua LP não carrega em 3s no 4G, você está pagando pra perder lead.

---

POST 9 — TLDR
TLDR:
→ Hero claro em 3 segundos
→ CTA específico
→ Prova social com dados reais
→ Formulário curto
→ Clareza > beleza
→ Urgência real, não fabricada
→ Mobile primeiro

---

POST 10 — QUESTION
Qual desses erros você mais vê no mercado? 👇

---

POST 11 — CTA
➡ Me siga pra mais conteúdo sobre CRO e conversão.
🔄 Compartilha o post 1 se alguém precisa ouvir isso.
```

## Notas para o Agente

1. **Nunca gere conteúdo sem antes consultar `references/diretrizes-eticas.md`**
2. Se o usuário pedir algo que viola uma diretriz ética, explique por quê e ofereça alternativa
3. Se o usuário não forneceu dados biográficos e o template exige, PERGUNTE — não invente
4. Priorize templates que não exigem dados pessoais se o contexto do usuário é desconhecido
5. "Viral" não é o objetivo — "viral E substancial" é. Um sem o outro não serve
6. O conteúdo deve ser publicável sem edição, mas o usuário DEVE revisar antes de publicar
7. Cada post individual deve ter no máximo 500 caracteres (limite do Threads)
