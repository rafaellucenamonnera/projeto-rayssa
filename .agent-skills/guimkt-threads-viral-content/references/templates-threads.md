# Templates de Thread — Referência Completa

Cada template segue a estrutura **Hook → Hold → Help** (H³).
Escolha o template baseado no subtipo e no input do usuário.

---

## TEMPLATE T1 — Thread de Lições (Subtipo: Professor)

### Quando usar
Usuário quer compartilhar aprendizados, erros, ou framework prático.

### Estrutura

```
POST 1 — HOOK
[Timeframe] fazendo [foco/nicho/profissão] me ensinou o que realmente funciona:

POST 2-N — VALUE (uma lição por post)
[Emoji numérico] [Título da lição — 2-4 palavras, começar com verbo]
[Frase 1: O QUE fazer — máx 25 palavras]
[Frase 2: POR QUE funciona — máx 25 palavras]

POST N+1 — TLDR
TLDR:
→ [Lição 1 — essência em 5 palavras]
→ [Lição 2 — essência em 5 palavras]
[...todas as lições...]

POST N+2 — QUESTION
Qual dessas lições bateu mais forte pra você? 👇

POST N+3 — CTA
➡ Me siga pra mais [tema] sem enrolação.
🔄 Compartilha o post 1 se fez sentido.
[CTA da oferta, se houver]
```

### Regras específicas
- 5-10 lições (números ímpares performam melhor)
- Ordenar: mais contrarian primeiro → mais convencional por último
- Cada lição entre 5-40 caracteres no título
- Incluir no mínimo: 1 dado real, 1 metáfora, 1 exemplo prático
- Bullets crescem do mais curto pro mais longo (ritmo visual)

### Variações de hook
- "[X] anos fazendo [Y] me ensinaram o que realmente funciona:"
- "[X] erros que cometi em [Y] — e o que aprendi com cada um:"
- "Se eu pudesse voltar [X] anos, diria isso pro meu eu mais novo:"

## TEMPLATE T2 — Thread Biográfica (Subtipo: Narrador)

### Quando usar
Usuário tem história real de transformação pessoal/profissional.

**⚠️ ATENÇÃO ÉTICA: TODOS os dados biográficos devem vir do usuário.**

### Estrutura

```
POST 1 — HOOK
Tenho [IDADE].
Depois de [ANOS] de [DOR/EXPERIÊNCIA], aqui vão [NÚMERO]
[termo para lições] sobre [TEMA] que eu queria ter sabido
[TEMPO ATRÁS].

POST 2 — CONTEXT (a virada)
[Descrição breve da situação antes]
[O que mudou / o ponto de virada]

POST 3-N — VALUE (lições da jornada)
[Emoji numérico] [Título]
[O que aprendi — frase curta]
[Por que importa — frase curta]

POST N+1 — PAYOFF (reflexão)
[Contraste antes x depois]
[Frase de encerramento que conecta toda a narrativa]

POST N+2 — TLDR
[Resumo das lições]

POST N+3 — QUESTION
[Pergunta de engajamento]

POST N+4 — CTA
[Stack de CTAs]
```

### Inputs obrigatórios do usuário
| Input | Exemplo | Substituto se não fornecer |
|-------|---------|---------------------------|
| Idade | "32" | **NÃO GERE. Pergunte.** |
| Anos de experiência/luta | "8 anos" | **NÃO GERE. Pergunte.** |
| Dor/experiência | "agências que não entregam" | **NÃO GERE. Pergunte.** |
| Situação atual | "fundei minha própria agência" | **NÃO GERE. Pergunte.** |

### Regras específicas
- Vulnerabilidade real, não performática
- A história deve ter arco: situação ruim → aprendizado → transformação
- NUNCA inventar elementos dramáticos que não aconteceram
- Se a história do usuário é "simples", respeite — não dramatize artificialmente

## TEMPLATE T3 — Thread de Curadoria (Subtipo: Sintetizador)

### Quando usar
Usuário quer sintetizar insights de múltiplas fontes, tendências ou análises.

### Estrutura

```
POST 1 — HOOK
[Dado/insight surpreendente que abre o tema]
Aqui vai o que [fonte/análise/experiência] revela sobre [tema]:

POST 2-N — VALUE (insights curados)
[Emoji numérico] [Insight condensado]
[Contexto ou exemplo — 1-2 frases]
[Implicação prática — 1 frase]

POST N+1 — SYNTHESIS (a grande conclusão)
[Conecta todos os insights em uma tese central]

POST N+2 — TLDR
[Resumo dos insights]

POST N+3 — QUESTION + CTA
[Pergunta + stack de CTAs]
```

### Regras específicas
- Citar fontes reais sempre que possível
- Se o insight é opinião do autor, declarar como tal
- Conectar cada insight a uma ação prática
- Não listar insights desconectados — deve haver fio condutor

## TEMPLATE T4 — Thread Analítica (Subtipo: Analista)

### Quando usar
Usuário quer analisar dados, tendências ou benchmarks do mercado.

### Estrutura

```
POST 1 — HOOK
[Dado surpreendente ou contraintuitivo]
[Promessa de análise: "Aqui vai o que os números mostram:"]

POST 2 — CONTEXT
[Contextualiza o dado: de onde vem, por que importa]

POST 3-N — ANALYSIS (cada post = 1 ponto de análise)
[Dado ou observação]
[Interpretação]
[Implicação prática]

POST N+1 — CONCLUSION
[Tese central que conecta toda a análise]

POST N+2 — TLDR + QUESTION + CTA
[Resumo → Pergunta → CTAs]
```

### Regras específicas
- TODOS os dados devem ser reais e fornecidos pelo usuário ou verificáveis
- Se é estimativa, declare: "Estimativas sugerem que..."
- Incluir contexto comparativo (benchmark, histórico, concorrente)
- Tom mais analítico, menos provocador — mas ainda com personalidade

## TEMPLATE T5 — Thread a Partir de URL

### Quando usar
Usuário fornece URL e quer transformar em thread original.

### Estrutura
- Escolher subtipo (Sintetizador ou Professor) baseado no conteúdo fonte
- Extrair 5-9 insights principais
- Reinterpretar com a voz do usuário
- Creditar fonte: "Inspirado por [título/autor] — minha interpretação:"

### Regras específicas
- NUNCA copiar frases da fonte original
- Reinterpretar, não resumir
- Adicionar perspectiva e experiência do usuário quando possível
- Creditar a fonte de forma visível e honesta
