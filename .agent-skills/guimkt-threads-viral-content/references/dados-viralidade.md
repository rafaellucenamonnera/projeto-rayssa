# Dados de Viralidade — Cheat-Sheet de Performance

Dados derivados da análise de 307+ posts virais no Threads.
Use como referência interna para decisões de estrutura e formato.

**⚠️ NOTA ÉTICA**: Quando estes dados forem INSERIDOS no conteúdo que o usuário
vai publicar, qualifique: "análises de posts virais sugerem que..." — nunca

## Regras de Formatação Baseadas em Dados

### Leiturabilidade
- Mediana dos posts virais: nível de leitura de 4º ano (Flesch-Kincaid)
- Clareza vence complexidade. Sempre
- Frases ≤ 25 palavras. Parágrafos ≤ 3 linhas

### Tamanho
- Thread ideal: 8-12 posts, 700-900 palavras totais
- Post ideal: 60-100 palavras por post individual
- Cada post ≤ 500 caracteres (limite da plataforma)

### Emojis
- Ponteiros direcionais (👇 ➡) presentes em 95% dos hits — manter esparsos
- Máximo 3 emojis por thread inteira
- NUNCA usar 🚀 ou 🔥 (marcadores de conteúdo genérico/spam)
- Emojis numerados (1️⃣-🔟) são aceitáveis para listas

### Estrutura comprovada
```
HOOK (Post 1)
    → Para o scroll. Estabelece credibilidade + promessa
VALUE (Posts 2 a N-3)
    → Entrega as lições/insights do mais contrarian ao mais convencional
PAYOFF (Post N-2)
    → A lição final, geralmente a mais universal/emocional
TLDR (Post N-1)
    → Cheat-sheet de todas as lições em uma lista compacta
QUESTION (Post N)
    → Pergunta aberta que convida resposta pessoal
CTA (Post N+1)
    → Stack de 3 CTAs: follow → share → oferta
```

## 10 Arquétipos de Hook

| # | Arquétipo | Por que funciona | Template |
|---|-----------|-----------------|----------|
| 1 | Idade/Experiência | Credibilidade instantânea | "Tenho [idade]. Depois de [X anos] em [campo], aqui vai o que ninguém te ensina:" |
| 2 | Momento Definidor | Vulnerabilidade gera curiosidade | "[Data]: tudo desmoronou — [evento]. O que aconteceu depois mudou tudo:" |
| 3 | Resultado Concreto | Promessa de insight interno | "[Número grande] de [resultado] em [período] — [N] jogadas que fizeram acontecer:" |
| 4 | Verdade Dura / Contrarian | Provoca emoção e debate | "Se prepara: [claim contrarian]. Deixa eu te mostrar os dados:" |
| 5 | Preview de Lista | Payoff claro, alavanca números | "[N] lições que eu tatuaria no meu eu de [X] anos atrás 👇" |
| 6 | Chamada de Público | Personaliza e filtra instantaneamente | "Se você é [avatar] e está [dor], lê isso antes de [ação] de novo:" |
| 7 | Empatia / Ponto de Dor | Rapport via luta compartilhada | "É difícil pra caramba [dor compartilhada]. Eu também sentia isso — até [verbo] esse caminho:" |
| 8 | Como Fazer | Orientado a solução, alto valor percebido | "Como [resultado desejado] sem [obstáculo] — vou [verbo] o mapa:" |
| 9 | Fato Surpreendente | Desperta curiosidade com dados | "[Fonte confiável] descobriu que [estatística]. Aqui vai como [verbo] esse insight:" |
| 10 | Cena Vívida | Joga o leitor dentro de uma história | "[Detalhe sensorial]. Percebi [verdade] — agora vou [verbo] pra você:" |

**Pro-tip:** Combine arquétipos para mais punch (ex: Idade + Contrarian + Lista).

## Padrões de CTA que Convertem

### Stack de 3 CTAs (para threads)
```
➡ Me siga pra [benefício claro de seguir].
🔄 Compartilha o post 1 se [razão pra compartilhar].
[Emoji] [CTA da oferta com URL ou "link na bio"].
```

### CTA simples (para posts)
```
➡ Me siga pra mais [tema] direto ao ponto.
```

### Regras de CTA
- Pedir compartilhamento explicitamente triplica reposts
- CTA com benefício claro > CTA genérico
- Máximo 3 CTAs por thread (mais = dispersão)
- CTA de follow primeiro, share segundo, oferta por último
- Se não tem oferta, omitir terceiro CTA (não inventar)

## Formatação Visual para Máximo Impacto

### Ferramentas de formatação aceitáveis
Estes caracteres especiais podem ser usados como bullets ou divisores:

**Setas e ponteiros:** → ➡ ← ↑ ↓ ➤ ▸ ►
**Marcadores:** ✦ ▪ ▫ ★ ☆ ✓ ✔
**Estruturais:** — (em dash), ... (reticências), // (quebra temática)

### Uso de whitespace
- Linha em branco entre cada bloco de pensamento
- Frase sozinha em linha própria = ênfase
- Não amontoar texto — o espaço em branco é parte do design

### O que NUNCA usar
- Hashtags (nunca, em nenhum formato)
- Emojis excessivos (máx 3 por thread)
- 🚀 🔥 (marcadores de conteúdo spam)
- Formatação markdown (bold, itálico) — Threads não renderiza
