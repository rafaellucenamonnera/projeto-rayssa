## 1. Processo LPO em 5 Etapas

### Etapa 1: Project Brief
- Escopo e overview do projeto
- Stakeholders, pessoas e departamentos que precisam aprovar ou ser consultados
- Objetivos de negócio
- Fonte de tráfego, landing pages atuais e anúncios
- Recursos disponíveis e entregáveis claramente definidos

### Etapa 2: Research
- Insights quantitativos iniciais (device, sample size, taxa de conversão, bounce rate)
- Landing page experience walkthrough (funil completo)
- Qualitativo: entrevistas CS/Sales/Usuários, heatmaps, scroll maps, review mining, feedback polls
- Deep dive quantitativo: funnel analysis, flow analysis, browsers, sample sizes
- Overview e priorização de findings

### Etapa 3: Landing Page Treatment
- Crie um "diagnóstico geral" e plano de otimização: o que está errado e como melhorar (baseado na pesquisa)
- Obtenha clareza das mudanças propostas, categorize em: Estrutura, Copy e Design
- Faça um outline rough da copy
- Comece a montar wireframe / mockup

### Etapa 4: Approval & Implementation
- Reserve 1-2h para reunião com stakeholders e decision makers
- Apresente os principais findings da pesquisa, diagnóstico geral e walkthrough do wireframe/mockup
- Planeje próximos passos: colaboração com copywriter e designer
- Time de dev coloca a LP live e pronta

### Etapa 5: Experimentation & Validation
- Se tiver tráfego suficiente: teste A/B para validar
- Alternativa: compare períodos (experimentação sem A/B)
- Aprenda continuamente com experimentos e pesquisa ongoing
- Valide o máximo possível para facilitar aprendizado real

## 2. Fast & Slow Thinking (Kahneman) Aplicado a LPO

### Sistema 1 (Pensamento Rápido)
- Intuitivo, automático, sem esforço
- Modo padrão do cérebro
- Toma decisões por atalhos mentais (heurísticas)
- "Máquina de saltar para conclusões"

### Sistema 2 (Pensamento Lento)
- Analítico, deliberado, requer esforço
- "O controlador preguiçoso" — só ativa quando necessário
- Engajá-lo requer energia do usuário

### Implicações para LPO:
- **Design para Sistema 1:** Layout limpo, hierarquia visual clara, informações escaneáveis
- **Minimize ativação do Sistema 2:** Reduza carga cognitiva, simplifique decisões
- **Framing importa:** A forma como você entrega a mensagem impacta diretamente a percepção (ex: "90% de sobrevivência" vs. "10% de mortalidade")
- **WYSIATI (What You See Is All There Is):** O que está visível na tela é tudo que o usuário considera para sua decisão. Se uma informação crucial não está visível, ela não existe para o usuário.

### Carga Cognitiva:
- Excesso de informação na tela → sobrecarga → abandono
- Hierarquia visual ruim → usuário não sabe onde olhar → confusão
- CTAs conflitantes → paralisia de decisão

## 3. 5 Níveis de Consciência — Detalhado

### 1. Unaware (Menos Consciente)
- **Estado:** Não tem consciência do problema ou necessidade
- **Conteúdo:** Educar sobre o problema, não sobre a solução
- **Erro comum:** Página vai direto para "Compre agora"
- **Volume de conteúdo:** Alto — precisa educar muito

### 2. Problem-Aware
- **Estado:** Sabe do problema, não conhece soluções
- **Conteúdo:** Empatizar com a dor e introduzir o conceito da solução
- **Erro comum:** Página mergulha em features específicas
- **Volume de conteúdo:** Alto

### 3. Solution-Aware
- **Estado:** Conhece soluções existentes, não conhece seu produto
- **Conteúdo:** Apresentar produto como a solução, destacar diferenciais e benefícios
- **Erro comum:** Página foca em pricing quando precisa mostrar por que é a escolha certa
- **Volume de conteúdo:** Médio-alto

### 4. Product-Aware
- **Estado:** Conhece seu produto mas não está convencido
- **Conteúdo:** Convencer com valor, aumentar motivação, endereçar barreiras e objeções
- **Erro comum:** Página é genérica e high-level quando precisa de especificidade
- **Volume de conteúdo:** Médio

### 5. Most-Aware (Brand-Aware)
- **Estado:** Totalmente consciente e informado, precisa do empurrão final
- **Conteúdo:** Ofertas especiais, lembretes, cases, incentivos
- **Erro comum:** Página introduz problema e solução do zero
- **Volume de conteúdo:** Baixo — direto ao ponto

### Relação Inversamente Proporcional:
- Quanto MENOS consciente → MAIS informação necessária, MENOR probabilidade de conversão
- Quanto MAIS consciente → MENOS informação necessária, MAIOR probabilidade de conversão

### Complexidade do Objetivo de Conversão:
- **Simples (baixo risco, baixo preço):** Ex: capa de celular de R$30. Pouca informação necessária.
- **Complexo (alto risco, alto preço):** Ex: software de R$50k/ano. Muita informação, reassurance, provas.

## 4. 3 Tipos de Fricção do Usuário — Detalhado

### Fricção de Interação
**Definição:** Problemas práticos ao interagir com a landing page.

**Fontes comuns:**
- Experiência com bugs; elementos de UI que não funcionam
- Layout responsivo que não adapta a device/screen size
- Overlays e ofertas intrusivas
- Experiência de signup/checkout complicada
- Páginas lentas / tempo de carregamento alto

**Exemplos reais:**
- Elementos core de UI escondidos
- UI e interação contra-intuitivas
- Overlay cobrindo elemento core de UI
- CTA sticky escondido atrás de overlay promovendo app

### Fricção Cognitiva
**Definição:** O usuário precisa gastar esforço mental extra para entender a landing page.

**Fontes comuns:**
- Layout/design issues: wall of text, fonte muito pequena/grande, conteúdo demais em uma tela
- Hierarquia visual ruim
- Uso confuso de imagens, cues direcionais, ícones sem sentido
- UI e navegação complicadas
- Copy e messaging confusos, jargão, linguagem técnica

**Exemplos reais:**
- Escolhas ruins de layout/design com baixo contraste
- Background dominante interfere com UI e copy
- Erros de hierarquia visual (CTA invisível, texto ilegível)
- Múltiplos overlays intrusivos com CTAs irrelevantes competindo por atenção

### Fricção Emocional
**Definição:** Emoções negativas que impedem o usuário de dizer "sim".

**Fontes comuns:**
- Dark patterns causando desconfiança
- Falar para o nível de consciência errado
- Falhar em reforçar motivações e endereçar objeções
- Falhar em responder perguntas críticas
- Pedir informação demais cedo demais
- Reação emocional a fricção de interação e cognitiva

**Exemplos reais:**
- LP que não corresponde às expectativas do anúncio
- Preço apresentado de forma enganosa (ex: destaque de R$29 quando o real é R$199)
- Uso manipulativo de "commitment" e "cancel"
- Messaging vago com CTA genérico

### Como usar na pesquisa e apresentação:
- Categorize TODOS os findings em uma das 3 categorias
- Apresente para stakeholders usando esta taxonomia — é intuitiva
- Use para priorizar: fricção de interação geralmente tem quick wins, fricção emocional requer mudanças mais profundas

## 5. Pesquisa Quantitativa

### GA4 — Core Insights para LPO
- **Users, conversions (key events), conversion rate:** Volume e taxa base
- **Bounce Rate:** Quantos usuários saem sem fazer nada?
- **Device category:** Split e performance por device
- **Source medium:** Fontes de tráfego e performance por fonte
- **Page path exploration:** Que páginas os usuários visitam após a LP?
- **Funnel exploration:** Onde no funil estão os dropoffs?

### GA4 — Reports e Explorations:
1. Standard landing page report
2. Free form exploration para LPO
3. Path exploration
4. Funnel exploration

### Page Speed Insights
- Performance real de páginas em mobile e desktop
- Tempo de carregamento e problemas de performance
- Core Web Vitals, diagnósticos e sugestões
- Imagens grandes, código pesado, scripts de terceiros
- Link: pagespeed.web.dev/

## 6. Pesquisa Qualitativa — 5 Métodos Core

### Método 1: Análise Heurística / Full Experience Walkthrough
**Propósito:** Familiarizar-se com a experiência completa e identificar áreas de otimização.

**Processo:**
1. Comece pelo anúncio e percorra cada passo da experiência
2. Use as perguntas como guia e anote ao longo do caminho

**Perguntas — Primeira Impressão (15 segundos):**
- Qual sua primeira impressão da landing page?
- Como ela te faz sentir?
- Parece uma continuação relevante do ad/fonte de tráfego?
- O que chama atenção?
- Parece confiável?
- Quais são suas expectativas neste ponto?

**Perguntas — Análise Detalhada:**
- O conteúdo está estruturado em ordem lógica?
- Há algo confuso?
- Você tem perguntas não respondidas?
- Há algo para adicionar, remover ou reorganizar?
- Há problemas de UI/UX ou bugs?
- Quais são suas expectativas neste ponto?

**Notas:** É o começo, não a análise final. Ponto de partida para pesquisa mais profunda.

### Método 2: Entrevistas com CS, Sales e Account Executives
**Propósito:** Obter insights cruciais de quem fala com quem compra.

**Abordagem:**
- Sessões de 30 minutos (presencial ou call)
- Envie convite explicando propósito
- Grave a sessão para não precisar anotar tudo

**Perguntas para Customer Success:**
1. Quais são as perguntas/preocupações mais comuns dos usuários?
2. Como você tipicamente ajuda a endereçar essas questões?
3. Pode dar exemplos de feedback positivo?
4. Há aspectos do produto que usuários acham difícil de entender/usar?
5. Há algo no site atual que você mudaria?
6. Como explicaria brevemente o produto/serviço para um amigo?
7. Perdi algo? Quer adicionar alguma coisa?

**Perguntas para Sales Reps & Account Executives:**
1. Quais são as perguntas mais comuns durante demos/calls de venda?
2. Quais aspectos geram mais entusiasmo nos prospects?
3. Quais são as objeções mais comuns?
4. Como você tipicamente endereça essas objeções?
5. Há algo que você gostaria que prospects soubessem ANTES do call?
6. Há algo no site atual que você mudaria?
7. Como explicaria brevemente o produto/serviço para um amigo?
8. Perdi algo? Quer adicionar alguma coisa?

**Dicas:** Ouça mais do que fala. Evite perguntas direcionadoras. Foque em motivações, fricção emocional e perguntas-chave.

### Método 3: Heatmaps (Scroll, Click, Atenção)
**Propósito:** Insight sobre como usuários interagem com a LP.

**O que observar:**
- Quão longe os usuários estão scrollando
- O que estão clicando/tappando
- Quais elementos recebem mais atenção
- Padrões de engajamento

**Notas:**
- Use como guia, não como verdade absoluta
- Quanto mais cliques, melhor (amostra maior = mais precisão)
- Ótima ferramenta para comunicar com designers
- Procure padrões (ex: itens mais clicados em FAQ)
- Lance heatmaps junto com novas versões da LP

### Método 4: Feedback Polls (Sniper Surveys)
**Propósito:** Insight claro e acionável com mínimo incômodo ao usuário.

**Perguntas inspiradoras:**
- Qual é o principal propósito da sua visita hoje?
- Você tem alguma pergunta não respondida neste ponto?
- Você visitou outros [tipo de serviço/produto]?
- O que é mais importante para você? (apresente 3 opções)
- Houve algo que quase te impediu de comprar hoje? (página de confirmação)

**Melhores práticas:**
- Curto e simples: 1 pergunta por vez
- Perguntas claras e específicas
- Tipos: open-ended, múltipla escolha, híbrido
- Considere trigger após delay (ex: 20 segundos)
- 100+ respostas é uma amostra decente
- Use com parcimônia

### Método 5: Review Mining & Analysis
**Propósito:** Usar reviews de clientes como fonte de dados de VOC (Voice of Customer).

**O que buscar:**
- Pain points, motivações, problemas resolvidos
- Benefícios, top features
- Linguagem usada, descrições
- Sentimento geral

**Fontes:** Trustpilot, Google Reviews, G2, Capterra, Amazon, App Store. Reviews de concorrentes também são úteis.

**Usando IA para análise:**
- Scrape ou copie reviews manualmente
- Faça upload para IA (PDF ou direto)
- Perguntas úteis:
  - Quais são os principais temas?
  - O que os reviewers dizem sobre [tema]?
  - O que os usuários mais gostam/não gostam?
  - Quais são as principais áreas de melhoria?
  - Que palavras os reviewers usam para descrever?
  - Pode mostrar alguns quotes?
- NÃO peça números ou estatísticas à IA para isso

## 7. Estrutura e Componentes de LP — Detalhado

### Componentes Essenciais:
- Headline / proposta de valor
- Subheadline
- Lista de bullets (benefícios)
- Imagem / vídeo
- Call-to-action
- Elemento de confiança (trust badges, logos)
- Depoimento / review
- Seções de conteúdo (features, benefícios, como funciona)
- Carrossel de conteúdo (overview rápido de aspectos importantes)
- Accordion (FAQ / mais informações)

### Modelos de Estrutura:

**Modelo Compacto:**
1. Headline + Subheadline + Imagem/vídeo + CTA
2. Trust element
3. Content carousel
4. Content section
5. CTA
6. Accordion

**Modelo com Social Proof:**
1. Headline + Bullet Points + Imagem/vídeo + CTA
2. Reviews
3. CTA
4. Content section
5. Content carousel

**Modelo Completo:**
1. Headline + Subheadline + CTA + Imagem/vídeo
2. Trust element
3. Content section
4. CTA
5. Reviews
6. Content carousel

**Modelo Extenso (produto complexo):**
1. Headline + Bullet Points + CTA + Trust element
2. Content Section 1-3
3. Image
4. Content Section 4-6
5. Trust Element
6. Reviews/carousel
7. Accordion
8. CTA final

### Hierarquia de Informação:
- Conteúdo mais importante → topo
- Conteúdo menos importante → base
- Primeiro screenfull é CRUCIAL: resumo, pontos-chave, proposta de valor

### Primeiros 2 Screenfuls — Checklist:
- [ ] Super limpo e claro
- [ ] Reforça motivação
- [ ] Responde perguntas-chave
- [ ] É confiável e crível
- [ ] Baixa fricção
- [ ] Caminho claro para conversão

### Princípio Fundamental:
Faça sua pesquisa LPO PRIMEIRO. Depois comece a construir wireframes. Você provavelmente vai trabalhar com um designer na página final.
